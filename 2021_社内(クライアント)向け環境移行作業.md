# 開発・実装内容

【概要】  
レガシー環境で稼働している社内ツール群を対象に、移行プロジェクトに参画。
移行対象となる約300ツールについて、移行前後の動作を考慮した改修およびテストを担当した。

【どのような機能の開発・実装か】
- 既存ツール移行対応
- UTF-8対応改修
- PHP8対応改修
- PostgreSQL対応改修
- 動作検証およびテスト

【課題・問題点】  
既存環境から新環境への移行に伴い、文字コード、OS、PHP、データベースが大きく変更されるため、
既存ツールに多数の影響が発生することが想定された。

また、移行対象が約300ツールと多く、効率的な調査・改修が求められた。

【打ち手・使用した技術】

■チーム対応  
- SJISからUTF-8への移行対応
- Windows ServerからLinux環境への移行対応
- PHP5.3系からPHP8系への移行対応
- MySQLからPostgreSQLへの移行対応
- HTML / CSS / JavaScript / jQueryによる既存画面改修

■個人対応  
- 移行対象ツールの調査および影響範囲確認
- UTF-8化対応の実施
- PHPバージョン、MySQL、PostgreSQL差異による不具合対応
- エンドユーザーとの仕様調整
- 工数見積および進捗管理
- 新人教育および移行手順書作成

【思考プロセス】  
新環境での影響、もともと作成者不在で確かな仕様がわからないものもあったため、
改修後の自身での動作確認、利用チームのクライアントに協力を仰ぎ、極力致命的なバグが起きないように留意した。

またこの作業の過程で仕様書を作成するようにチームに促し、今後の安全も図った

【成果】
- 約300ツールを対象とした環境移行対応を実施
- UTF-8化およびPHP8対応を実施し、新環境への移行を支援
- 移行に伴う不具合調査および改修を担当
- サブリーダーとして進捗管理および新人教育を実施
