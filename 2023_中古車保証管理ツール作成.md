# 開発・実装内容

【概要】  
中古車保証管理業務を効率化するための管理ツール開発プロジェクトに参画
詳細設計から実装、動作テストまで担当した

【どのような機能の開発・実装か】
- 中古車保証管理機能
- 画面開発

【課題・問題点】  
基本設計書のみが存在し、テーブル定義やER図、詳細処理設計など実装に必要な情報が不足していたため、
そのままでは開発を進められない状況だった。

【打ち手・使用した技術】

■チーム対応  
- Laravel 10を利用したWebアプリケーション開発
- MySQLによるデータ管理
- HTML / CSS / JavaScript / jQueryによる画面開発
- Tailwind CSSによるUI実装
- Dockerによる開発環境構築
- GitHubによるソースコード管理

■個人対応  
- 画面設計書をもとに不足している設計内容を洗い出し
- テーブル定義、ER図、詳細処理設計についてクライアントへ提案および確認を実施
- 詳細設計の作成および実装対応
- 社内テストドキュメント作成
- 動作テスト実施および改修対応

 【思考プロセス】  
上流はクライアントの担当だったが、仕様書として完成といわれたものが、基本設計のみで詳細処理などが不明だった。
ただ時間もないため、開発するにあたり、最低限必要なテーブル定義、ER図、詳細処理設計などを一旦必要最低限用意してもらうように考え、補足しつつ動くようにしていました。

【成果】
- 不足していた詳細設計、テーブル定義、ER図の必要性を洗い出し、クライアントへ提案
- 実装に必要な設計情報を整備し、開発を進行可能な状態へ改善
- 中古車保証管理ツールの実装およびテストを実施
- 詳細設計から実装、テストまで一貫して担当
